# 操作日志处理规则

## 规则文件说明
本文件定义了操作日志的自动化管理规则，用于指导 AI 助手按标准流程处理操作日志。

## 基本原则

### 1. 时间获取规则
- 严格按照 `TIME_RULES.md` 规则获取当前时间
- 使用命令：`php -r "echo date('Y-m-d H:i:s');"`
- 禁止使用硬编码日期

### 2. 日志拆分规则
- 当天记录保留在主 `operateLog.md` 文件中
- 历史记录按日期拆分到 `logs/operateLog-by-ymd/` 目录下对应文件
- 文件命名格式：`YYYY-MM-DD.md`
- 特殊文件：`YYYY-MM-d0.md` 用于记录项目开始前的基础工作

### 3. 文件结构规范

#### 主文件 (operateLog.md) 结构：
```markdown
# 操作日志

## 历史记录说明
历史操作记录已按日期拆分到 logs/operateLog-by-ymd 目录下的对应文件中：
- YYYY-MM-d0.md：项目基础架构开发阶段
- YYYY-MM-DD.md：具体日期的操作记录

---

## [当天最新记录]
[具体内容]

---

*最后更新：[当前时间]*
*历史记录已拆分到 logs/operateLog-by-ymd/ 目录*
```

#### 拆分文件结构：
```markdown
# 操作日志 - YYYY-MM-DD

## [具体记录内容]

---

*拆分自原 operateLog.md 文件*
*拆分时间：[拆分时的当前时间]*
```

## 自动化处理流程

### 每日执行步骤（Quest Mode）

1. **获取当前时间**
   ```bash
   php -r "echo date('Y-m-d H:i:s');"
   ```

2. **分析待处理记录**
   - 读取 `../operateLog.md` 文件
   - 识别不是今天日期的记录
   - 按日期分组待拆分内容

3. **执行拆分操作**
   - 为每个历史日期创建对应的 `.md` 文件
   - 将内容移动到对应的日期文件中
   - 保持记录完整性和格式一致性

4. **更新主文件**
   - 保留今天的记录在主文件中
   - 添加历史记录说明部分
   - 更新最后修改时间

5. **验证和检查**
   - 确认所有历史记录已正确拆分
   - 验证主文件只包含今天的内容
   - 检查拆分文件的完整性

## 特殊情况处理

### 1. 错误时间修正
- 如发现错误的时间戳（如1月22日实际是项目开始阶段），应修正后合并到 d0 文件
- 在 d0 文件中添加修正说明

### 2. 文件冲突处理
- 如果目标文件已存在，应合并内容而不是覆盖
- 保持时间顺序，避免重复内容

### 3. 内容完整性
- 确保每个记录包含完整的上下文信息
- 保留原有的格式和链接
- 添加适当的拆分说明
